トリボナッチ数列第n項を求める際の計算量

ただ再帰関数を用いる場合
①n = 5 とすると、n = 4, n = 3, n = 2 全て求めることになる。
すると、②n = 4 で n = 3, n = 2, n = 1を求めることになるが、
① に出てくる n = 3 の計算に、②の n = 3の結果は使われない。
よって、計算量が激増する。

結果をメモ化する場合
①n = 5 とすると、n = 4, n = 3, n = 2 の値が必要になる
その後、②n = 4 で ③n = 3, n = 2, n = 1を求めることになり、
③の時点で n = 3 値が算出され、メモ化される
ruby風だと、
memo ={
  3 => 1
}
みたいな。
すると、①の n = 3 の計算では、すでに値が出ているので計算をすることはない。
このように、各校に対して1度のみ計算を行うため、計算量はO(N)となる。
